SLM Documentation

Overview

This documentation goes over usage for phase 1 of the new Site Log Manager 2.0 on slmdemo.igs.org. This is the second version of the original IGS SIte Log Manager (slm.igs.org) built in (2007?). Steps in this documentation are subject to change. To use the Site Log Manager 2.0, view the simple 4 step process below.

4 Step Process

  1. Select Station: Select a station from the list at the left. (If the Station you need is not there you will need to fill out the Propose a New IGS Site form or contact IGS for help)
  2. Edit Data: Once you select a station you will then be redirected to the “Edit” tab of the station with forms for each site log section. Select the form you would like to add/update metadata to and fill out the fields.
    • Upload Data (Optional): There is also the ability to upload files instead of manually entering in data. To upload a file: select “Upload Logs” in the top menu bar and click any of the download options
  3. Correct Data: Once the data is either manually entered or a sitelog file is uploaded, the system will let you know if there are any errors or missing data. This will be indicated by the Error Badge with a description of the error underneath the field. Make any corrections where needed. There are tooltips on the right of each field to help guide you.
  4. Submit Data: Once the data has been corrected select the “Review” tab to view your changes and click the “Publish” button at the top of the page. After the data has been visually validated you will receive an email of approval and that the data has been posted to IGS's main databases ready for use.
  5. Download Data (Optional): As you wait for the site log to be approved by the network coordinator, you can download your modified site log or download any of the past site logs by going to the “Downloads” tab.

About SLM

The IGS Site Log Manager (SLM) is a web based online application designed for the purpose of managing the metadata of IGS GNSS ground-based sites. The SLM utilizes a relational database with a new and improved web-based User Interface that allows for uploading of the traditional IGS ASCII Logs and/or Manual Data Entry. The SLM Validates the data against a set of criteria defined at: Site Log Instructions

With this new version, our primary goal is to maximize the reliability, accuracy, and searchability of site log metadata information. How to get there?

  • Easy Moderation and Review
  • Automation
    • Validation
    • Usage (Structured APIs)
  • Structured Data
    • Self-validating interchange formats (GeodesyML)
    • Easy to use renderings (JSON)
  • Broad Adoption
    • Open Source
    • Extendable

What Changed

  • Brand new user interface
    • Colors and fonts were updated to match the IGS Brand.
    • Cleaned up old, broken, and duplicate links
    • Restructured pages/sections for a better streamlined flow between system features
  • Dashboard
    • View the latest alerts for all of your stations and take action on any moderator requests
    • View recent station updates
  • Station Legend
    • Colors updated to be more accessible
    • *Subject to change*
  • Alerts/Notifications [New]
    • Get notified when there is an error in your site log that needs correction or any other action is needed.
      • Note: You will also receive an email
    • Alerts/Notifications will also appear on the dashboard and on the notifications page
  • Improved Editing
    • Update past sub-sections [New]
      • See an error in a past sub-section? Added the ability to update other sub-sections (i.e. antennas, receivers)
    • Improved Date Picker
      • New modern date picker UI
      • Allows for manual date/time entry
    • Keep track of last published data
      • When editing a field, a note underneath the field will show the last published data and show an icon on the field indicating it has been updated
  • Improved Validation/Moderation System
    • When updates are saved there will be a flag on each field and an alert on top that lets you know. The colors will also change to unpublished.
    • 3 main levels of validation flags
      • Highest level- anything that doesn’t match the format whatsoever won’t let you input data in the database, will reject it outright
      • Middle level- won’t let you publish until you fill in a certain field
      • Lowest level- you can still publish it but the flag is there because we encourage you update it
    • Admin flags
      • Admins also have the ability add in flags to review/fix once it goes into moderation
  • Download
    • Modified/Unpublished Log [New]
      • Download modified/unpublished .log version of your site log before it gets approved by the Network Coordinator
    • GeodesyML (Coming Soon)
    • JSON (Coming Soon)
  • Review and Submit (Previously View/Diff)
    • The View/Diff screen is now located in the “Review” tab
    • Changes are shown on one page instead of two
    • Can use the date picker to view and compare your changes to past site logs
  • Activity Log (Previous Change Log)
    • View a history of changes/updates to the site all in one page
    • Recent station updates can also be viewed on the Dashboard
  • Map View [New]
    • View stations in an all new interactive map with global view
    • See list of stations, key metadata, and links to download options in a table
    • Sort/Filter through stations and metadata

Development phases

  • In order to best meet the needs of IGS Member Agencies and to make the management of the station metadata as easy, accurate and complete as possible the IGS SLM will continue to be developed and improved.
  • This version is only the first Phase of the new SLM 2.0 and focuses on the core functionality of the old SLM including:
    • Edit/Moderation/Publish Functionality
    • Automated validation infrastructure
    • Basic API functionality
    • Site log upload/download
  • Additional phases will include improvements and brand new features, including but not limited to:
    • More robust automated validation
    • Full support for other file formats, such as GeodesyML (.xml) and JSON
    • Interoperability between other station metadata editing systems in collaboration with IGS Member Agencies through Fully searchable APIs
  • We welcome any comments or suggestions regarding your use of the system. Please submit any comments or suggestions to our contact form
